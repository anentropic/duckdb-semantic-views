  <behavior>
    - Self-reference: `from_alias == to_alias` returns Err containing "cannot reference itself"
    - Cycle detection: A->B->C->A returns Err containing "cycle detected in relationships" with path
    - Diamond detection: A->B, A->C, B->D, C->D returns Err containing "diamond" and "two paths to"
    - Orphan table: declared in tables but not in any relationship and not base table returns Err containing "orphan"
    - FK/PK count mismatch: join with 2 fk_columns referencing table with 1 pk_column returns Err with count mismatch
    - Unreachable source_table: dim with source_table="x" where "x" is not reachable from root returns Err
    - Toposort of valid tree A->B->C returns [A, B, C] (root first)
    - Toposort deterministic: same graph with different declaration order produces same output
    - Legacy defs (all joins have empty fk_columns): validate_graph returns Ok and skips validation
    - Single-table defs (no joins at all): validate_graph returns Ok
    - Error messages include fuzzy "did you mean?" suggestions using suggest_closest from expand.rs
  </behavior>
  <action>
Create `src/graph.rs` with:

1. `RelationshipGraph` struct with:
   - `edges: HashMap<String, Vec<String>>` — adjacency list (from_alias -> to_alias)
   - `reverse: HashMap<String, Vec<String>>` — reverse adjacency (to_alias -> from_aliases)
   - `all_nodes: HashSet<String>` — all declared table aliases (lowercased)
   - `root: String` — base table alias (first table in TABLES clause, lowercased)

2. `RelationshipGraph::from_definition(def: &SemanticViewDefinition) -> Result<Self, String>`:
   - Extract root from `def.tables.first()` (lowercased)
   - Build all_nodes from `def.tables` aliases (lowercased)
   - Iterate `def.joins` where `!join.fk_columns.is_empty()`:
     - Self-reference check: `from_alias == to_alias` (case-insensitive) -> immediate Err
     - Add edge from_alias -> to_alias (both lowercased)
     - Add reverse edge to_alias -> from_alias

3. `toposort(&self) -> Result<Vec<String>, String>` — Kahn's algorithm:
   - Compute in-degree for each node
   - Seed queue with root first (in-degree 0), then other zero-in-degree nodes in sorted order for determinism
   - BFS: pop node, decrement in-degree of neighbors, enqueue when in-degree reaches 0
   - If order.len() != all_nodes.len(), remaining nodes form a cycle -> format error with path naming: "cycle detected in relationships: X -> Y -> Z -> X"
   - To find cycle path: from any unvisited node, follow edges until revisiting a node

4. `check_no_diamonds(&self) -> Result<(), String>`:
   - For each node in `reverse`, if node != root and parents.len() > 1 -> Err with "diamond: two paths to 'X' via 'Y' and 'Z'"

5. `check_no_orphans(&self) -> Result<(), String>`:
   - An orphan is a non-root node with in-degree 0 AND out-degree 0 (not in edges keys, not in reverse keys)
   - Err with "orphan table 'X' is not connected by any relationship" plus fuzzy suggestion

6. `check_fk_pk_counts(def: &SemanticViewDefinition) -> Result<(), String>`:
   - For each join with non-empty fk_columns, find referenced table (join.table) in def.tables
   - If referenced table has non-empty pk_columns and fk_columns.len() != pk_columns.len() -> Err

7. `check_source_tables_reachable(def: &SemanticViewDefinition, graph: &RelationshipGraph) -> Result<(), String>`:
   - Collect all source_table aliases from dims and metrics
   - For each, verify it's in graph.all_nodes
   - If not found, error with "unknown source table 'X'" plus fuzzy suggestion against all_nodes

8. `pub fn validate_graph(def: &SemanticViewDefinition) -> Result<RelationshipGraph, String>`:
   - If no joins have non-empty fk_columns -> return Ok with empty/default graph (legacy skip)
   - If def.tables.is_empty() -> return Ok (legacy path, no tables declared)
   - Build graph via from_definition
   - Run: toposort(), check_no_diamonds(), check_no_orphans(), check_fk_pk_counts(), check_source_tables_reachable()
   - Return Ok(graph) if all pass

Add `pub mod graph;` to `src/lib.rs` (after `pub mod expand;`).

Use `crate::expand::suggest_closest` for fuzzy suggestions in error messages. Normalize all alias comparisons to lowercase using `to_ascii_lowercase()`.

All unit tests go in `#[cfg(test)] mod tests` inside graph.rs.
