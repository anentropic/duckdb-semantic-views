**duckdb_type → LogicalTypeHandle mapping:**
```rust
use duckdb::core::{LogicalTypeHandle, LogicalTypeId};
use libduckdb_sys as ffi;

fn type_from_duckdb_type(t: u32) -> LogicalTypeHandle {
    // Complex/invalid types fall back to VARCHAR
    match t {
        ffi::DUCKDB_TYPE_DUCKDB_TYPE_INVALID
        | ffi::DUCKDB_TYPE_DUCKDB_TYPE_LIST
        | ffi::DUCKDB_TYPE_DUCKDB_TYPE_STRUCT
        | ffi::DUCKDB_TYPE_DUCKDB_TYPE_MAP
        | ffi::DUCKDB_TYPE_DUCKDB_TYPE_ENUM => {
            LogicalTypeHandle::from(LogicalTypeId::Varchar)
        }
        // DECIMAL falls back to DOUBLE (avoids width/scale complexity)
        ffi::DUCKDB_TYPE_DUCKDB_TYPE_DECIMAL => {
            LogicalTypeHandle::from(LogicalTypeId::Double)
        }
        _ => LogicalTypeHandle::from(LogicalTypeId::from(t)),
    }
}
```

**bind() current flow:**
```
1. Extract view_name + dimensions + metrics
2. Validate at least one requested
3. Look up catalog JSON
4. Parse SemanticViewDefinition from JSON
5. expand() → expanded_sql
6. infer_schema_or_default() → column_names (names only, types discarded)
7. bind.add_result_column(name, Varchar) for each column
```

**bind() Plan 03 changes:**
```
After step 4 (def now has column_types_inferred):
  4a. Build requested_column_names from dimensions + metrics (same order as expand output)
  4b. If def.column_types_inferred.len() == column count:
        use stored types (convert u32 → LogicalTypeHandle via type_from_duckdb_type)
      Else:
        fall back to VARCHAR for all columns
  Replace step 7: bind.add_result_column(name, typed_handle) for each column
  Store Vec<LogicalTypeId> (or LogicalTypeHandle) in SemanticViewBindData for func()
```

Note: `infer_schema_or_default()` was used to get column names. Plan 03 still needs the names.
The column count from `column_types_inferred` was built from ALL dims+metrics at DDL time (Plan 02).
At query time, only a subset of dims+metrics may be requested. So `column_types_inferred` cannot
be used positionally against the query output directly. Instead:
- At bind time, build the requested columns list (dimensions requested + metrics requested)
- Call `try_infer_schema` on `state.conn` (the QUERY connection, safe in bind) with the LIMIT 0 expanded SQL
  to get column names AND types for the specific subset requested
- The existing `infer_schema_or_default` already does this! It just discards types.
- Plan 03: modify `infer_schema_or_default` (or replace it) to return both names AND types.

Revised approach (simpler than using column_types_inferred positionally):
- At bind time, run `try_infer_schema` on the PER-QUERY expanded SQL (already done implicitly)
- Use returned types directly to declare columns (no stored type lookup needed)
- `column_types_inferred` on the definition is only used when `try_infer_schema` fails (fallback)

This is actually the cleanest approach — bind-time inference on the query connection is already
confirmed safe (RESEARCH.md: "Bind-time inference on state.conn... per-query overhead. Either works").
The stored types serve as a fallback only.

IMPORTANT: This approach means the per-query LIMIT 0 overhead remains. That is acceptable for v0.2.0.
The stored `column_types_inferred` is the fallback when per-query inference fails.

    fn type_from_duckdb_type_u32(t: u32) -> LogicalTypeHandle {

        const STRUCT: u32 = 23;  // DUCKDB_TYPE_DUCKDB_TYPE_STRUCT
        const MAP: u32 = 25;     // DUCKDB_TYPE_DUCKDB_TYPE_MAP
        match t {
            INVALID | ENUM | LIST | STRUCT | MAP => {
                LogicalTypeHandle::from(LogicalTypeId::Varchar)
            }
            DECIMAL => LogicalTypeHandle::from(LogicalTypeId::Double),
            _ => LogicalTypeHandle::from(LogicalTypeId::from(t)),
        }
    }

    Rename `infer_schema_or_default` to `infer_schema_with_types` and change its return type to `(Vec<String>, Vec<u32>)`:
    ```rust
    fn infer_schema_with_types(
        conn: ffi::duckdb_connection,
        expanded_sql: &str,
        dimensions: &[String],
        metrics: &[String],
        def: &SemanticViewDefinition,
    ) -> (Vec<String>, Vec<u32>) {
        let limit0_sql = format!("{expanded_sql} LIMIT 0");
        let inferred = unsafe { try_infer_schema(conn, &limit0_sql) };

        if let Some((names, types)) = inferred {
            let type_ids = types.iter().map(|t| *t as u32).collect();
            return (names, type_ids);
        }

        // Fallback: derive names from definition, all types are 0 (INVALID → VARCHAR)
        let mut column_names = Vec::new();
        for dim_name in dimensions {
            let canonical = def
                .dimensions
                .iter()
                .find(|d| d.name.eq_ignore_ascii_case(dim_name))
                .map_or_else(|| dim_name.clone(), |d| d.name.clone());
            column_names.push(canonical);
        }
        for met_name in metrics {
            let canonical = def
                .metrics
                .iter()
                .find(|m| m.name.eq_ignore_ascii_case(met_name))
                .map_or_else(|| met_name.clone(), |m| m.name.clone());
            column_names.push(canonical);
        }
        let type_ids = vec![0u32; column_names.len()]; // INVALID → VARCHAR fallback
        (column_names, type_ids)
    }
    ```

    The new func() structure:
    ```rust
    fn func(func: &TableFunctionInfo<Self>, output: &mut DataChunkHandle) -> Result<(), Box<dyn std::error::Error>> {
        let init_data = func.get_init_data();
        if init_data.done.swap(true, Ordering::Relaxed) {
            output.set_len(0);
            return Ok(());
        }

        let bind_data = func.get_bind_data();
        let state = unsafe { &*func.get_extra_info::<QueryState>() };

        // Execute the expanded SQL directly (no VARCHAR cast wrapper needed for typed columns).
        let mut result = unsafe {
            execute_sql_raw(state.conn, &bind_data.expanded_sql)
                .map_err(|e| QueryError::SqlExecution {
                    expanded_sql: bind_data.expanded_sql.clone(),
                    duckdb_error: e,
                })?
        };

        let col_count = unsafe { ffi::duckdb_column_count(&mut result) } as usize;
        let chunk_count = unsafe { ffi::duckdb_result_chunk_count(result) } as usize;

        // Collect total row count first for sizing output.
        let mut total_rows = 0usize;
        // Collect all row data per column, typed.
        // Use a column-major layout for efficient typed copy.
        let mut col_data: Vec<Vec<TypedValue>> = vec![Vec::new(); col_count];

        for chunk_idx in 0..chunk_count {
            let chunk = unsafe { ffi::duckdb_result_get_chunk(result, chunk_idx as ffi::idx_t) };
            if chunk.is_null() { continue; }
            let row_count = unsafe { ffi::duckdb_data_chunk_get_size(chunk) } as usize;
            total_rows += row_count;

            for col_idx in 0..col_count.min(bind_data.column_type_ids.len()) {
                let type_id = bind_data.column_type_ids[col_idx];
                // Read this column's values for this chunk and append to col_data[col_idx]
                unsafe {
                    read_typed_column(chunk, col_idx, row_count, type_id, &mut col_data[col_idx]);
                }
            }
            unsafe { ffi::duckdb_destroy_data_chunk(&mut { chunk }); }
        }

        // Write col_data to output.
        if total_rows > 0 {
            for col_idx in 0..col_count.min(bind_data.column_names.len()) {
                let type_id = if col_idx < bind_data.column_type_ids.len() {
                    bind_data.column_type_ids[col_idx]
                } else { 0 };
                write_typed_column(output, col_idx, &col_data[col_idx], total_rows, type_id);
            }
        }
        output.set_len(total_rows);
        unsafe { ffi::duckdb_destroy_result(&mut result); }
        Ok(())
    }
    ```
