# Phase 65.1 Plan 12: WR-09 Parser-Hook Idempotence Summary

**Closed WR-09 by adding an iterator-based dedup check inside `sv_register_parser_hooks` that guards the entire `ParserExtension` build + `Register` block (including the `SemanticViewsParserInfo` allocation) when a matching `sv_parser_override` function pointer is already present in `DBConfig::GetCallbackManager().ParserExtensions()`. Repeat `LOAD semantic_views` in the same process now leaves the parser-extension list with exactly one entry instead of accumulating one entry per LOAD. Behavioural coverage by `test/integration/test_load_extension_twice_idempotent.py` (LOAD-twice + sentinel DDL through the parser_override hook); structural coverage by `tests/parser_hook_idempotent.rs` (six-assertion guard against the dedup code shape + the new `sv_count_parser_extensions` public helper added per the B-07 plan-checker fix).**

## Accomplishments

- **Idempotence guard landed in `sv_register_parser_hooks`** (cpp/src/shim.cpp:2658-2729). The pattern:
  ```cpp
  auto &config = DBConfig::GetConfig(db);
  auto &cbmgr = config.GetCallbackManager();
  bool already_registered = false;
  for (auto &existing : cbmgr.ParserExtensions()) {
      if (existing.parser_override == sv_parser_override) {
          already_registered = true;
          break;
      }
  }
  if (!already_registered) {
      // Build ParserExtension, allocate SemanticViewsParserInfo,
      // Register, then SetOption("allow_parser_override_extension").
  }
  ```
  Critical: the `SemanticViewsParserInfo` allocation is inside the guard, so the skip path does not leak a fresh parser_info shared_ptr (the very leak called out in RESEARCH lines 706-710).

- **Compat header extended** (cpp/include/parser_extension_compat.hpp). Added a minimal forward-declaration for `ExtensionCallbackRegistry`, the `ExtensionCallbackIteratorHelper<T>` template (with inline `begin()`/`end()` iterators), and two extra methods on `ExtensionCallbackManager`: `ParserExtensions() const` and `HasParserExtensions() const`. The existing file header documents the ODR contract: declarations must mirror duckdb.cpp exactly. Only the methods we link against need to be declared; the constructor/destructor for the iterator helper are non-inline in the amalgamation and we only receive instances by value from `ParserExtensions()`.

- **Public structural-verification helper added** (cpp/src/shim.cpp:2792-2860, cpp/src/shim.hpp:48-89). `sv_count_parser_extensions(db_handle, error_buf, error_buf_len) -> int32_t` iterates `cbmgr.ParserExtensions()` and counts entries whose `parser_override == sv_parser_override`. Returns -1 on failure with the message in `error_buf` (WR-02 ABI convention). The helper's docstring at the declaration site cites the option-(a) decision rationale + the B-07 plan-checker fix.
