<objective>
Introduce the `BorrowedConnection` newtype (CONTEXT D-10) and migrate all 17 read-side dispatchers enumerated in `65.1-RESEARCH.md` to consume it. The FFI signature stays `conn: libduckdb_sys::duckdb_connection` (the C++ side passes a raw handle); the newtype is constructed inside the function body via `let borrowed = unsafe { BorrowedConnection::new(conn) };`. Anywhere downstream that needs the raw handle reaches into `.as_raw()`. The defining property: `ffi::duckdb_disconnect` takes `ffi::duckdb_connection` not `BorrowedConnection`, so the call simply does not type-check.

Extend `tests/no_long_lived_conn.rs` with a second AST walker (`DisconnectFinder`) that recurses through all of `src/**/*.rs` looking for any call expression whose last path segment is `duckdb_disconnect`. Defence in depth alongside the type-level enforcement (D-11).

This plan is entirely Rust-side and does NOT modify any C++ shim. It can run in parallel with all other Wave 1 plans.

Purpose: Eliminate the "code review only" enforcement of the BORROW contract. The per-call `Connection probe(*context.db)` constructed in C++ bind callbacks is a stack object — `duckdb_disconnect` on its bridged handle would `delete` a stack pointer (UB). Today only ~20 `reinterpret_cast<duckdb_connection>(&probe)` sites and code comments protect this; a contributor could regress trivially.

Output: ~17 dispatcher signatures updated (parameter wrap pattern); 1 new newtype + ~3 helper-function signature changes; 1 AST guard extension; 1 compile_fail doctest.

Implements D-10, D-11 (WR-05).
</objective>

pub struct BorrowedConnection(ffi::duckdb_connection);

impl BorrowedConnection {
    #[must_use]
    pub unsafe fn new(conn: ffi::duckdb_connection) -> Self { Self(conn) }
    #[must_use]
    pub fn as_raw(&self) -> ffi::duckdb_connection { self.0 }
    #[must_use]
    pub fn is_null(&self) -> bool { self.0.is_null() }
}
// Deliberately NO Drop, NO Clone, NO impls that could lead to disconnect.
