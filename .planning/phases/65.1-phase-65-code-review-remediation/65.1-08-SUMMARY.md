## Decisions Made

- **Reword the IN-01 comment to avoid quoting the historical stale phrase verbatim.** The plan's acceptance criterion `grep -n "flip the registration" cpp/src/shim.cpp` returning zero matches would have failed if I had quoted the original recommendation verbatim for context. Rewrote to use the paraphrase "rewire registration through the init_local callback" so future readers still understand what was retired without re-introducing the grep hit. Acceptance verified: `grep -n "flip the registration" cpp/src/shim.cpp` returns no matches (exit 1).
- **`throw InternalException` rather than `BinderException` for the null-state branch.** Null `local_state` despite `init_local` being wired is a `cannot happen` invariant violation вЂ” `InternalException` is the DuckDB type for this category. `BinderException` would mislead callers into thinking they wrote a malformed query. The catch site at the parser/executor boundary surfaces both exception types verbatim, so users still see the underlying message text.
- **Verbatim function-name strings in the InternalException message** rather than the `__FUNCTION__` / `__func__` C macros. Compiler-derived names can include namespace prefixes, return-type signatures (depending on toolchain), or C++ mangling. The literal string keeps the diagnostic deterministic and grep-able. Acceptance criterion confirmed: `grep -nE "local_state missing despite init_local registration" cpp/src/shim.cpp | wc -l` returns 4.
- **Sandbox bypass for `just test-sql` and `just test-all`.** The macOS sandbox blocks `mktemp` writes to `/var/folders/.../T/` (the path is hardcoded inside macOS libc and ignores `$TMPDIR`). First `just test-sql` run failed with `mktemp: mkstemp failed ... Operation not permitted` on the sqllogictest runner setup. Retried with `dangerouslyDisableSandbox: true` per the CLAUDE.md Rule 2 pre-approval for build/test commands; both `just test-sql` and `just test-all` then ran cleanly to completion. This matches the project memory ("Sandbox quirk" entry).
