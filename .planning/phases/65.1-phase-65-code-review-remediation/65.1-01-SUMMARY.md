## Decisions Made

- **Added `pytest>=7.0` to PEP-723 dependencies of both Python stubs.** The plan literally specified `pytest.skip(...)` as the body, but `test/integration/` has no precedent for pulling pytest — every existing integration test calls `sys.exit()` directly. Two options: (a) replace `pytest.skip` with a print + `sys.exit(0)` shim (deviates from plan wording), or (b) honour the plan wording and add pytest to the inline metadata so `uv run` can satisfy the import (Rule 3 auto-fix — blocking dependency). Chose (b) because the plan author was deliberate about the skip semantic and a future pytest-driven test runner would see the SKIPPED outcome correctly.
- **Kept all real assertions in commented-out blocks at the END of each stub** rather than embedding them as `#[ignore]` or `statement error` placeholders. Two reasons: (1) keeps the stubs visibly "scaffold, not test", reducing the risk of someone reading the stub and thinking the coverage already exists; (2) the implementing plan's diff becomes "uncomment + adjust" which is easier to code-review than "search file for where to insert".
